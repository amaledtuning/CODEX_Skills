# Improve Readability Information Design

Use this skill to make information easier to read and scan across media. It applies to text-heavy UI, slides, PDF one-pagers, documents, dashboards, instructions, posters, cards, labels, and image prompts that include text or structured information.

## Workflow

1. Identify the reading mode: glance, scan, compare, learn, decide, or long-form reading.
2. Read `references/principles.md` when the task involves text density, type hierarchy, line length, contrast, labels, information layout, or slide/document structure.
3. Diagnose readability blockers: tiny type, all caps, weak hierarchy, long paragraphs, poor contrast, crowded layout, unclear labels, or missing chunks.
4. Redesign the information structure before polishing visuals: headings, grouping, hierarchy, rhythm, line length, and whitespace.
5. Pair with execution skills as needed: `presentations`, `documents`, `frontend-design`, `react`, `imagegen`, `figma`, or `spreadsheets`.

## Output Expectations

Return concrete copy/layout guidance: what to shorten, split, promote, demote, relabel, convert to bullets, convert to a table, or move into progressive disclosure.

## Boundaries

Do not use this skill for pure visual hierarchy without a readability problem. Use `shape-visual-attention` for attention routing and `reduce-cognitive-load` for multi-step comprehension or memory burden.
