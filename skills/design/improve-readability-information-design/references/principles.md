# Readability and Information Design Principles

Source anchor: paraphrased from the design book in `D:\Project Skills\Книги по дизайну\100_glavnyh_principov_dizayna_kak_uderzhat_vnimanie.pdf`, especially typography, screen reading, text chunking, contrast, case, and line-length sections around pp. 42-58.

## Core Principles

- Optimize for the reader's task. Glance, scan, compare, and study modes need different density and hierarchy.
- Use mixed case for normal reading. All caps can work for short labels, but long all-caps text slows recognition.
- Use adequate size, line height, and contrast before adding decorative type treatments.
- Break long text into chunks. Headings, bullets, short paragraphs, tables, and callouts reduce reading strain.
- Match line length to purpose. Long lines can support speed in some contexts; shorter lines usually improve comfort and comprehension for dense material.
- Put the most useful words at the start of headings, labels, and bullets.
- Treat whitespace as structure. It is not empty space; it tells the reader where one idea ends and another begins.
- Make hierarchy semantic, not just decorative. The visual structure should match the information structure.

## Practical Checklist

- Define the reader's next action after reading.
- Convert walls of text into headings, bullets, numbered steps, tables, or callouts.
- Make headings specific enough to scan without reading the body.
- Keep paragraph width comfortable for the medium.
- Raise contrast when text carries operational or decision-critical information.
- Remove unnecessary capitalization, centered paragraphs, and decorative type that slows scanning.
- Align related labels and values so comparisons are easy.
- For slides, prefer one clear idea per slide or section.
- For PDFs and documents, create a repeatable hierarchy for title, section, body, notes, and calls to action.
- For images/posters, avoid text that must be read at a size the final medium will not support.

## Anti-Patterns

- Dense paragraphs on a slide.
- Tiny footnote-style text used for important information.
- All caps for sentences or paragraphs.
- Weak heading hierarchy where everything looks equally important.
- Labels that are visually separated from the data they explain.
- Low contrast body text on textured or busy backgrounds.

## Output Pattern

When using this skill, produce:

1. Reading mode and reader goal.
2. Main readability blockers.
3. Revised information structure.
4. Typography and spacing guidance.
5. Medium-specific implementation notes for slide, PDF, document, UI, image, poster, or infographic.
