# Onboarding and Mental Model Principles

Source anchor: paraphrased from the design book in `D:\Project Skills\Книги по дизайну\100_glavnyh_principov_dizayna_kak_uderzhat_vnimanie.pdf`, especially mental models, conceptual models, examples, memory, and storytelling sections around pp. 65-66 and 90-100.

## Core Principles

- People bring prior assumptions. New products are interpreted through existing categories, habits, and metaphors.
- A product needs a clear conceptual model. If the model is hidden, users invent one, often incorrectly.
- Teach only what is needed for the next useful action. Exhaustive onboarding creates load before value.
- Use concrete examples. People learn unfamiliar systems faster when abstract rules are tied to visible examples.
- Use narrative sequence for complex ideas. A simple story can make cause, context, and consequence easier to remember.
- Confirm learning through action. Let users do something meaningful, then show feedback that proves the model.
- Use empty states as teaching moments. Empty spaces should explain purpose, next action, and expected future value.
- Repeat important concepts in different forms when long-term recall matters.

## Practical Checklist

- State the likely wrong or incomplete user assumption.
- Name the intended conceptual model in one sentence.
- Define the first successful user action.
- Replace abstract feature tours with task-based examples.
- Use before/after, cause/effect, or step/result structure.
- Keep vocabulary consistent across UI, slides, PDF, and help copy.
- Use progressive disclosure: concept first, detail second, edge cases later.
- In explainer decks, make each slide answer one question in the learning path.
- In product visuals, show the object, the action, and the result together when possible.

## Anti-Patterns

- Feature tours that list controls without explaining why they matter.
- Empty states that only say there is no data.
- Onboarding that asks for commitment before showing value.
- Metaphors that conflict with how the product actually behaves.
- Explainers that introduce internal terminology before user goals.
- Walkthroughs that cannot be skipped or resumed.

## Output Pattern

When using this skill, produce:

1. Existing user assumption.
2. Intended mental model.
3. First useful action.
4. Teaching sequence with examples.
5. Feedback or proof that confirms understanding.
6. Medium-specific notes for onboarding UI, deck, PDF, empty state, tutorial, or visual explainer.
